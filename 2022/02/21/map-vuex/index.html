map_vuex

2022-02-21

项目开发的时候，遇到多个视图都使用地图的情况，如果每个页面都init一个map，似乎太重复和冗余，但写一个map通过组件引入的方式，又会遇到调用另一个组件内的方法的问题。因此，趁机学习一下vuex。

Vuex：
状态管理模式。集中式存储管理所有组件的状态。
单个组件，单向数据流，很好理解。
多个组件共享状态时，容易破坏单向数据流：

多个视图依赖于同一状态。
来自不同视图的行为需要变更同一状态。

vuex：全局单例模式管理。

首先，一个简单的store模式：
单独写一个store，state中记录值，写方法变更值，外面的组件调用state获得值，调用方法改变值。

回到vuex：
核心是store（仓库），内包含state（状态）：
store中的state变化，相应的组件中的state都会变化。
不能直接改变state，想改变state，只能通过 提交 (commit) mutation。【这样的好处是可以跟踪状态的变化，可以通过console.log(‘say something…’)记录变化之类的】

state：
可以在每个组件中引入，但麻烦。
直接在根中注入：写在app里。

mapState 辅助函数：
一个组件需要获取多个状态。

Getter：
从 store 中的 state 中派生出一些状态。

Mutation： 同步事务
非常类似于事件：每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 回调函数 (handler)。这个回调函数就是我们实际进行状态更改的地方，并且它会接受 state 作为第一个参数。

Action：异步

Module： store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块。

一个插曲：
我的问题：
项目开发的时候，遇到多个视图都使用地图的情况，如果每个页面都init一个map，似乎太重复和冗余，但写一个map通过组件引入的方式，又会遇到调用另一个组件内的方法的问题。
比如我的项目中，tab2页面需要对map进行搜索，定位，checkin页面需要展示被tab2定位后的状态的map。
目前是通过传参实现的：checkin是tab2的子页面，从tab2跳转checkin的时候，poi的经纬度作为参数传递。
目前，有一个用来引入amap的locationMap组件。tab2引入locationMap，可以显示地图，但当tab2调用AMap.placeSearch方法时会报错，因为tab2和locationMap是兄弟节点。只能在tab2又写了initMap来引入地图，和locationMap重复了。
因此，尝试下vuex保存地图状态，把方法写在vuex中。

也就是说，目前是tab2重写ininMap()，tab2/checkin依赖传参。
我的解决：将LocationMap写作tab2的子组件，通过this.$refs.locationmap获取到map和placeSearch，从而实现地图上搜索poi的功能。
