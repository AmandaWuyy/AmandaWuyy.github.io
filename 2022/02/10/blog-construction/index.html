个人博客搭建（Hexo + Github Pages）

2022-02-10

目前比较流行的三款静态网站生成器：Jekyll / Hugo / Hexo 比较

出于构建速度快、在GitHub Pages部署简单、支持中文的优势，选择Hexo

前提：

安装Node.js和git，npm已经使用淘宝镜像（快一些）

（一）Github配置

创建仓库，用于存放博客文章，用户名必须是GitHub名，也就是说和前面的Owner一致

用户名.github.io

生成SSHkey

如果以前配置过：找到C:\Users\wyyhc.ssh中的pub后缀文件id_rsa.pub，复制
否则：

ssh-keygen -t rsa -C "注释文字（可以为邮箱或者名字）"    //这将按照你提供的注释文字，创建一对密钥

生成后到相应的位置复制

将本地生成的SSH秘钥添加到GitHub：

点击GitHub头像 → settings → SSH and GPG keys → New SSH key
自定义Title，粘贴key

验证SSH Keys是否配置成功

ssh -T git@github.com

完善github个人信息

$ git config --global user.name "用户名" 
$ git config --global user.email "邮箱"

（二）安装Hexo

全局安装Hexo脚手架

 npm install hexo-cli -g

新建文件夹，用于存放hexo，并安装部署插件。本文创建ownblog文件夹

cd ownblog
npm init    //生成一个package.json文件
npm install hexo-deployer-git --save    //安装部署插件
npm install    //安装依赖

初始化Hexo

hexo init

每次编译运行

hexo g    //生成网页，先生成网页才能预览运行
hexo s    //启动一个本地服务器，实现本地预览，默认端口4000

执行以上命令之后，hexo就会在public文件夹生成相关html文件，在浏览器地址栏输入localhost:4000 ,即可预示例文章

（三）写一篇博客

新建.md文件

hexo new 'test-blog'    //ownblog\\source\\_posts下生成一个test-blog.md文件

打开test-blog.md文件编辑博客

生成html页面

hexo g

本地预览

hexo s

（四）部署到GitHub

在ownblog目录下执行命令

hexo d

打开浏览器，在地址栏输入 github用户名.github.io 预览效果

更换主题

Hexo官网主题

找到主题的GitHub链接，clone到themes文件夹，修改 ownblog/_config.yml 以切换主题，修改 themes/clean-blog/_config.yml 以更改主题设计

结束

以后每次到source的_posts文件夹下新建md文件，更新保存后

hexo g 
hexo s    //本地可预览
hexo d    //https://amandawuyy.github.io/可预览
