关于排序：
347、451:
为什么和215不同，这里建堆的时候shiftUp和shiftDown需要.value？
因为这里在使用MinHeap的时候，插入的是{ value, key }，那么在比较大小的时候，设定用value值来比较。
因此，在实际使用过程中，数据结构根据题意动态调整。
Map类的题，在堆中插入{ value, key }会方便一点。

为什么这里map和h可以用const声明？
因为const在声明Object类型的时候，里面的属性值可以改变。

关于Map：
两种迭代方式：
（1）for…of 迭代
（2）forEach() 迭代
forEach中只有一个参数的时候，该参数是value；
forEach中如果有两个参数的时候，需要注意它跟for…of的区别，forEach中的两个参数分别是value和key，它跟for…of是反的。

Map和Array的转换：

//Map转Array：
const array = Array.from(map);
//Array转Map：
const map = new Map(array);

注意：Map转换成数组是一个二维数组。同理要想一个数组能够转换成Map，该数组必须是二维数组。

75:
灵活思考。

关于贪心：
每次操作，局部最优；最终结果，全局最优。
435:
感觉逻辑是对的，只是超时，通过55/58：
（1）按头排序：从小到大头排序，头同，从小到大尾排序。
头同，则尾大删。
头不同，若新头<原尾，则尾大删。
（2）按尾排序：从大到小尾排序，尾同，从小到大头排序。
尾同，则头小删。
尾不同，若新尾<原头，则头小删。
反向思考。求最多的不重叠区间需要移除区间的个数，先求不重叠区间，再减。

435、452：
扎气球的，就按计算最多的不重叠区间做。
为什么尾排序？因为要计算最多的不重叠区间，区间的结尾最重要，结尾越小，留给后面的空间越大，越有可能不重叠。
